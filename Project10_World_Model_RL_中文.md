## 背景：Dyna-Q 的思想

在标准 Q-learning（Lec21）中，每一个真实环境步产生一条训练样本 `(s, a, r, s')`，用来更新 Q 表。学习之所以慢，是因为每一条信息都来自真实世界。

Dyna-Q（Lec17，基于模型的 RL 一节）增加了一步：在用真实样本更新之后，智能体还用一个**学到的世界模型**额外生成 **K 条想象的样本**，并在这些样本上也进行训练。于是每个真实步现在产生 **K+1** 次更新，而不是 1 次。

```
标准 Q-learning：               Dyna-Q：

真实步 → 1 次 Q 更新             真实步 → 1 次 Q 更新
                                       → 通过世界模型进行 K 次想象更新
```

世界模型其实就是从数据中学到的两个函数：

```
转移模型（Transition model）：  f(s, a) → 预测的下一状态 s'
奖励模型（Reward model）：       g(s, a) → 预测的奖励 r
```

如果这些预测是准确的，那么想象的样本几乎和真实样本一样好。如果预测不准确，想象的样本可能会**误导**智能体。本项目的实验正是设计来观察这两种情况的。

## 第 1 部分：构建环境（建议用时：约 4 小时）

### 网格世界

构建一个随机的 8×8 网格世界。这与 Lec18 中介绍的环境是同一类型，但增加了一个重要的东西：**噪声（noise）**。

```
网格：   8×8 个格子，位置用 (row, col) 索引，row, col ∈ {0,...,7}
起点：   左上角 (0, 0)，每个 episode 开始时位于此处
终点：   右下角 (7, 7)
墙：     在固定位置放置 8–10 个阻挡移动的墙格
        （设计时务必保证从起点到终点始终存在一条通路）
```

**随机性转移：** 当智能体采取动作 `a` 时，以概率 **p = 0.2** 环境会忽略 `a`，转而让智能体朝**均匀随机**的某个方向移动。以概率 0.8 执行其意图的动作。如果移动会撞墙或越界，则智能体**原地不动**。

**为什么要有随机性？** 一个确定性环境是平凡可预测的——世界模型只需记住一张查找表。随机性迫使世界模型去学习一个**真正的、关于结果的概率分布**，这才是有趣且贴近现实的情况。它同时也让 Q-learning 变得更难，从而为 Dyna-Q 的想象经验留出帮忙的空间。

```
动作：  0=上, 1=下, 2=左, 3=右   （4 个动作）

奖励：
  +1.0   到达终点 (7,7) —— episode 结束
  −0.01  其余每一步（小幅时间惩罚）
  若 100 步内未到达终点，episode 结束
```

**状态表示：** 状态是整数元组 `(row, col)`——共 64 个可能状态。这个规模足够小，使得 tabular Q-learning 基线可以完美工作，也足够小到你能把整个价值函数可视化为一张 8×8 热力图。

## 第 2 部分：构建世界模型（建议用时：约 5 小时）

世界模型由两个小型神经网络组成，它们在从真实环境采集的数据上训练。

### 第 1 步：采集训练数据

在训练世界模型之前，用**随机策略**（采取均匀随机的动作）采集一个真实环境转移的数据集：

```python
dataset = []
state = env.reset()
for _ in range(10_000):
    action = random.randint(0, 3)
    next_state, reward, done, _ = env.step(action)
    dataset.append((state, action, reward, next_state))
    if done:
        state = env.reset()
    else:
        state = next_state
```

使用随机策略而非最优策略是有意为之：你希望数据集能**覆盖整个状态空间**，而不只是最优策略会访问的那些状态。把数据集按 80/20 划分为训练集与验证集。

**为什么用随机策略？** 如果你用一个训练好的智能体来采集数据，它只会访问最优路径附近的状态。那样世界模型就永远见不到大部分状态空间，而从这些未探索状态出发的想象样本将会完全错误。

### 第 2 步：把状态和动作表示为向量

在喂给神经网络之前，状态 `(row, col)` 必须转换成向量。对全部 64 个网格位置使用简单的 one-hot 编码：

```python
def state_to_vector(state):
    # 状态：(row, col)
    idx = state[0] * 8 + state[1]     # 转成整数索引 0..63
    vec = np.zeros(64, dtype=np.float32)
    vec[idx] = 1.0
    return vec

def action_to_vector(action):
    vec = np.zeros(4, dtype=np.float32)
    vec[action] = 1.0
    return vec
```

两个世界模型网络的输入都是状态向量（64 个值）与动作向量（4 个值）的**拼接**——总共 68 个输入值。

### 第 3 步：训练转移网络

转移网络在给定当前状态和动作的情况下，预测下一状态。

```
架构：
  输入：   68 个值（状态 one-hot + 动作 one-hot）
  第1层：  Linear(68, 256) → ReLU
  第2层：  Linear(256, 256) → ReLU
  输出：   Linear(256, 64)    ← 对 64 个可能的下一状态的 logits
  损失：   CrossEntropyLoss(predicted_logits, true_next_state_index)
```

这是一个**分类**问题：给定 (s, a)，预测接下来是 64 个网格格子中的哪一个。这里用 CrossEntropy 损失是合适的——与 Lec11 中图像分类用的是同一种损失。

在训练集上训练 20 个 epoch。每个 epoch 之后，计算验证准确率（输出的 argmax 与真实下一状态相符的转移比例）。在这个环境上训练良好的模型应达到 **75–85%** 的验证准确率——**而不是 100%**，因为 20% 的噪声使得部分转移在本质上是不可预测的。

### 第 4 步：训练奖励网络

奖励网络在给定当前状态和动作的情况下，预测标量奖励。

```
架构：
  输入：   68 个值（状态 one-hot + 动作 one-hot）
  第1层：  Linear(68, 128) → ReLU
  第2层：  Linear(128, 64) → ReLU
  输出：   Linear(64, 1)     ← 标量奖励预测
  损失：   MSELoss(predicted_reward, true_reward)
```

训练 20 个 epoch。奖励预测应当比转移预测更容易——大多数转移给出 −0.01（高度可预测），只有进入 (7,7) 的转移给出 +1.0。

### 第 5 步：验证世界模型

在把世界模型用于规划之前，先验证它能正常工作：

- 挑选 5 个具体的 (state, action) 对。对每一个，查询转移网络并展示 **top-3 预测下一状态**（softmax 概率最高的三个）。这些预测在物理上合理吗？（例如，在格子 (3,4) 上执行 action=右，最有可能预测出 (3,5)，同时有一定概率落在 (3,4)，对应撞墙或噪声。）

## 第 3 部分：实现 Dyna-Q（建议用时：约 5 小时）

### 算法

Dyna-Q 用想象的经验来增强标准 Q-learning。Q 表的形状为 (64, 4)——每个 (state, action) 对一个值。

```python
def dyna_q(env, transition_model, reward_model,
           K=10, alpha=0.1, gamma=0.95,
           n_steps=50_000, epsilon_start=1.0, epsilon_end=0.05):

    Q = np.zeros((64, 4))           # Q 表
    replay_buffer = []              # 存放至今见过的真实转移

    state = env.reset()
    epsilon = epsilon_start

    for step in range(n_steps):
        # --- 真实环境步 ---
        epsilon = max(epsilon_end, epsilon_start - step / n_steps * (epsilon_start - epsilon_end))

        if random.random() < epsilon:
            action = random.randint(0, 3)
        else:
            action = np.argmax(Q[state_to_index(state)])

        next_state, reward, done, _ = env.step(action)

        # 真实 Q 更新（来自 Lec21 的标准 Q-learning）
        s  = state_to_index(state)
        s_ = state_to_index(next_state)
        Q[s, action] += alpha * (reward + gamma * np.max(Q[s_]) - Q[s, action])

        replay_buffer.append((state, action))    # 记住我们见过的
        if done:
            state = env.reset()
        else:
            state = next_state

        # --- 用世界模型进行 K 次想象步 ---
        for _ in range(K):
            # 从一个之前见过的 (state, action) 对中采样
            img_s, img_a = random.choice(replay_buffer)

            # 问世界模型会发生什么
            img_s_vec = concat(state_to_vector(img_s), action_to_vector(img_a))
            img_s_next_logits = transition_model(img_s_vec)
            img_s_next = argmax(softmax(img_s_next_logits))   # 采样下一状态

            img_r = reward_model(img_s_vec).item()

            # 想象的 Q 更新（与真实更新同一个公式）
            s_i  = state_to_index(img_s)
            s_i_ = img_s_next
            Q[s_i, img_a] += alpha * (img_r + gamma * np.max(Q[s_i_]) - Q[s_i, img_a])

    return Q
```

**一个关键设计决策：** 在采样想象的转移时，我们**只从智能体真正访问过的** `(state, action)` 对中采样。这可以防止世界模型在它从未见过的状态上被查询——那里它的预测将不可靠。这一点很重要：要在报告里解释它。

### 基线：普通 Q-learning

把标准 Q-learning 实现为 Dyna-Q 在 K=0 时的特例：

```python
def q_learning(env, alpha=0.1, gamma=0.95, n_steps=50_000, ...):
    return dyna_q(env, transition_model=None, reward_model=None, K=0, ...)
```

没有世界模型，没有想象步——纯粹的真实环境学习。

### 天花板：Value Iteration

用来自 `env.get_all_transitions()` 的**真实**转移概率实现 Value Iteration（Lec19）。这能在不经过任何试错学习的情况下给出最优 Q 值：

```python
def value_iteration(P, R, gamma=0.95, threshold=1e-6):
    # P[s, a, s'] = 转移概率
    # R[s, a]     = 期望奖励
    V = np.zeros(64)
    while True:
        V_new = np.max(R + gamma * np.sum(P * V[np.newaxis, np.newaxis, :], axis=2), axis=1)
        if np.max(np.abs(V_new - V)) < threshold:
            break
        V = V_new
    return V
```

Value Iteration 可以访问真实的环境动力学——无需学习。它代表**性能天花板**：任何数量的真实世界经验所能产生的最优策略。把你学到的智能体与它对比。

当 Dyna-Q 从这些未见过的对中采样一条想象转移时，它会生成一次假的 Q 更新。请用你自己的话描述：在许多这样的假更新之后，那个对的 Q 表项会发生什么。它会被推向一个高值吗？还是随机值？把它联系到 Q-learning 更新公式：

```
Q(s,a) ← Q(s,a) + α (r̂ + γ max Q(ŝ') − Q(s,a))
```

如果 r̂ 和 ŝ' 都是错的，那么 Q(s,a) 被拉向什么值？

**3. 坏模型在网格世界的什么地方失败得最厉害？** 对比三张 8×8 价值函数热力图：Value Iteration V*、好的 Dyna-Q、坏的 Dyna-Q。找出坏 Dyna-Q 的值与 V* 差异最大的 3 个格子。这些格子位于被那 200 条数据集覆盖良好的区域（起点附近），还是覆盖很差的区域（远离起点）？为什么覆盖度很重要？

**4. 量化数据集规模的影响。** 再做一个额外实验：在 **2000 条转移**（是坏模型的 10 倍，是好模型的 1/5）上训练世界模型，并用这个中等模型评估 Dyna-Q。把它的成功率曲线与其他条件画在一起。性能是随着数据增多而平滑提升，还是存在阈值效应？报告在 50000 真实步时、四种数据规模下的成功率：200、2000、10000 条转移，外加 K=0 基线。把它们做成柱状图（**图 5**）。

**5. 提出一个实际的修复方案。** 假设你处在一个真实应用中，只能采集到 200 条初始转移——数据采集确实很昂贵。在不预先采集更多数据的前提下，你能做些什么让世界模型更可靠？提出一个具体的技术方案。可考虑的选项：

- **不确定性感知规划（Uncertainty-aware planning）：** 只在模型有信心（对预测的下一状态有较高 softmax 概率）的地方使用世界模型来生成想象转移。丢弃那些模型不确定的想象转移。解释这会如何改变 Dyna-Q 算法。

- **保守想象（Conservative imagination）：** 按模型的预测置信度对想象 Q 更新加权——有信心的想象转移权重等同于真实转移；不确定的几乎不计权重。写出修改后的更新规则。

- **在线改进模型（Online model improvement）：** 不冻结世界模型，而是在 RL 训练过程中用新采集的转移继续训练它。解释这种做法的风险（提示：RL 策略会随时间改变，因此被访问状态的分布也在变——世界模型还能保持校准吗？）。

挑选一个选项，清晰地解释其机制，并论证为什么它在 200 条转移这个具体设定下会有帮助。

## 实用提示（Practical Tips）

**关于训练时间：** 世界模型训练很快——8000 条转移上跑 20 个 epoch 在 CPU 上不到 2 分钟。RL 训练（200000 步 × 3 种子 × 4 种条件 = 12 次运行）大致需要：

- CPU 上每次运行约 5 分钟（tabular Q-learning 很快）
- 全部运行总计约 1 小时算力

与 Project 1 不同，这里没有 GPU 瓶颈——一切都是 tabular 或小型 MLP。可以在一个下午内跑完所有实验。

**关于实现 Value Iteration：** `get_all_transitions()` 方法返回真实的 P[s,a,s'] 数组。Value Iteration 对一个 64 状态的 MDP 会在 1 秒内收敛。把它收敛得到的 V* 既用作图 3 的基线（在 200 个测试 episode 上评估其贪婪策略），又用作图 4 的真实色阶。

**关于失败分析的预测：** 在**运行任何东西之前**，把预测段落写在一个单独的文档里，或清晰标注在你失败实验脚本的顶部注释里。截图或带时间戳保存它。这是本项目中**最不可委托**的部分——AI 工具无法替你做这个预测，因为那时结果还不存在。

**关于调试一个「没有改善」的 Dyna-Q：** 如果 Dyna-Q 表现得和 Q-learning 一样，最常见的原因是：(1) 世界模型没训练好（检查训练时 loss 是否下降）；(2) 想象转移只从索引为 0 的状态采样（检查 replay buffer 采样）；(3) 函数调用里 K 被不小心设成了 0。
