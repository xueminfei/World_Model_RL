# World Model RL: Teaching an Agent to Imagine

Learning a neural world model for more sample-efficient reinforcement learning.

This project trains a neural-network world model on a noisy 8x8 gridworld and uses
it to generate imagined experience for the Dyna-Q algorithm. It then compares
Dyna-Q against plain Q-learning and Value Iteration to answer one central question:

> When a neural network learns to predict what happens next in an environment,
> does that prediction actually help an RL agent learn faster? And what happens
> when the prediction is wrong?

## Overview

Standard Q-learning learns only from real environment steps, which is slow when
each interaction is expensive. Dyna-Q (Sutton, 1990) augments each real step with
K imagined steps generated by a learned world model, turning one real transition
into K+1 Q-updates. This repository implements that idea end to end and studies
both the benefit of accurate predictions and the failure mode of inaccurate ones.

## Environment

- Grid: 8x8, state is an integer tuple `(row, col)`, 64 states total.
- Start: top-left `(0, 0)`. Goal: bottom-right `(7, 7)`.
- Walls: 8-10 fixed wall cells, with a guaranteed path from start to goal.
- Actions: `0=up, 1=down, 2=left, 3=right`.
- Stochastic transitions: with probability `p = 0.2` the intended action is
  ignored and a uniformly random direction is taken; otherwise the action is
  executed. Moving into a wall or off the grid keeps the agent in place.
- Reward: `+1.0` on reaching the goal (episode ends), `-0.01` on every other
  step. Episodes end after 100 steps if the goal is not reached.

Stochasticity is intentional: it forces the world model to learn a genuine
probability distribution instead of memorizing a lookup table, and gives Dyna-Q's
imagined experience room to help.

## World Model

Two small MLPs trained on 10,000 transitions collected with a random policy
(80/20 train/validation split), then frozen before any RL begins.

- Transition network (classification): `68 -> 256 -> 256 -> 64`, CrossEntropy
  loss, 20 epochs. Predicts which of the 64 cells comes next. Validation accuracy
  reaches 75-85%, not 100%, because the 20% noise makes some transitions
  genuinely unpredictable.
- Reward network (regression): `68 -> 128 -> 64 -> 1`, MSE loss, 20 epochs.

The network input is the concatenation of a one-hot state vector (64) and a
one-hot action vector (4), for 68 values total.

## Algorithms

- Dyna-Q (K=10): each real step is followed by 10 imagined Q-updates sampled from
  previously visited `(state, action)` pairs using the frozen world model.
- Q-learning (K=0): the no-world-model baseline, implemented as a special case of
  Dyna-Q.
- Value Iteration: the performance ceiling, computed from the true transition
  probabilities, representing the best policy any amount of experience could
  produce.

```
  gridworld.py        Environment (reset / step / get_all_transitions)

  world_model.py      Transition + reward networks, data collection, training

  plotting.py         Figure 1-5 generation
  main.py             Entry point, dispatches by stage
  results/            Saved metrics, figures, frozen model weights
  report/             Report drafts and pre-experiment prediction
```

## Experiments

- Main experiment: Value Iteration, Q-learning (K=0), and Dyna-Q (K=10) on the
  same gridworld; Q-learning and Dyna-Q each run with 3 random seeds, recording
  average reward and success rate every 500 steps.
- Forced failure analysis: a "bad" world model trained on only 200 transitions,
  plus a dataset-size sweep (200 / 2,000 / 10,000 transitions), to study how
  model error degrades learning.
